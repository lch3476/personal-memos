	1. 컴퓨터 구조

-컴퓨터는 전기의 on/off로 구성 -> 2진법

-집적도: 단위 면적에 트랜지스터 개수

-임베디드 - pc - 워크스테이션 - 서버 - 슈퍼컴퓨터

-CPU: 3, 머리
 -제어장치: cpu 내부 데이터, 컴퓨터 전체 데이터
  연산장치: 논리, 산술 연산
 주기억장치: 2, 4
 -RAM, ROM, Flash
 보조기억장치: 1
 -SSD, HDD, FDD, ODD 등
 출력장치: 5
 -키보드, 마우스 등

-도체, 부도체: 전기 통함, 안통함
 반도체: 조건부 도체

-전기는 열로 끝
 반도체 사용 - 전기 사용 - 열 - 과열 시 반도체 특성 무너짐 - 고장

-쿨러 쇠판은 열 전달용

-컴퓨터 성능은 부품간 밸런스 중요

-육안 점검 요소:
 팬 작동
 컨댄서(위, 아래 부품)
 핀: CPU핀 문제는 회생 불가
 나사 풀림
 접촉 불량
 반도체 크랙, 그을림
 

-추워서 안켜지면 온도 높이기

-RAM 접촉 불량:
 지우개로 청소, 뺐다 꼈다, bw-100, 무너진 핀 확인

-폰 노이만 구조: cpu는 주기억장치 0번 가서 명령 호출

-BIOS:
 self-check
 BIOS setting
 OS loading

-작업 시 정전기 조심

-모듈 단위 교체 시대. 효율적

-윈도우 업데이트 중 강제종료는 OS 손상 유발

-속도 향상:
 1. SSD + M.2
 2. 메모리 UP
 3. 시작 프로그램 관리 (msconfig)
 4. 미사용 프로그램 종료
 5. 디스크 관리
 6. 재부팅
 7. 요소 업그레이드
 8. OS 재설치

-메모리는 2개로 -> 듀얼 채널(문이 2개)

	2. 전기 기초

-전압: 물통, V
 전류: 물 흐름의 세기, A
 저항
 전력: 전압 * 전류

-교류: 전기 손실 및 변전 적음
 직류:
 어댑터: 교류를 직류로 변환

-Hz: 1주기(싸이클)이 1초에 몇번인지

-전원 장치 용량 계산:
 CPU + VGA + 메인보드(30) + HDD(10) + ODD(10) + SSD(5) + RAM(1)
 파워 표기 전력과 실출력 다름 주의
 1.5배 여유 두기
 전력 부족 시 예상치 못한 종료 혹은 윈도우 에러

-테스터기를 파워 메인 전력 핀에 사용함으로 접지 상태 확인 가능

-어댑터 규격은 기기에 맞추는게 좋음.

-IBM 메인 보드 규격: AT, ATX
 기계적 사항(인터페이스), 논리적 사항(핀의 역할), 전기적 사항(전기적 특성)

	3. BIOS

-BIOS는 ROM에 위치
 SW와 HW의 중간: 펌웨어 -> 주기억장치가 아닌 반도체(ROM) 안에

-임베디드: 프로그램을 짜서 반도체 안에 넣는 것

-기능:
 POST(Power On Self Test): CPU, VGA 등 정상 작동 확인. Beep 테스트로 에러 확인
 CMOS Setup S/W: 메인보드 하드웨어 조작 가능
 OS Loading

-부팅 시간 증가 -> 윈도우 로딩 전? -> 하드 디스크 불량 가능성 -> 하드에서 쇠구슬 딸깍 소리나면 수명 다한 것.

-연결 상태 확인 먼저:
 쿨러, SATA, 팬 작동

-퓨즈: 과전류로부터 부품 보호.

-연결이 브릿지 문제일 경우는 거의 없음:
 퓨즈 확인

-핫플러그인: 전원 on일 때 탈착 가능
 ps-2 단자는 전원 on일 때 탈부착 x -> 퓨즈 끊어짐

-반도체 종류:
 TTL
 CMOS: ROM옆에 RAM(CMOS 배터리로 on(리튬 이온 배터리))으로 CMOS 세팅 저장 -> 최근 부품은 flash 메모리로 배터리 불필요

-CMOS 배터리 방전, 배터리 납땜 깨짐 확인

-CMOS 초기화:
 배터리 빼기, 점프 스위치

-파티션: 칸막이
 포맷 : 의자, 책상 놓기

-화면x -> 모니터 드라이브
 소리x -> 사운드 드라이브

-HDD는 플래터와 헤드로 데이터 저장 -> 높은 플래터 RPM -> 접촉시 손상 -> 물리적 베드 섹터 -> 복구 불가
 논리적 베드 섹터는 partition 설정으로 해결 가능성 있음

-1. Low Level Format
 물리적인 상태 - OS 못봄
 2. Partition
 Window 설치 드라이브는 가장 빠른 곳으로
 3. High Level Format 
 클러스터 생성 -> 논리 섹터로

-OS 설치 전 체크리스트:
 데이터 백업, USB(혹은 ODD), 드라이브, APP

-OS 설치 전 키보드, 마우스 제외,  장치 모두 제거

-설치 시 업그레이드 옵션은 지양. 깔끔히 정리 안됨

-OS 설치 오류:
 HDD배드 섹터
 하드웨어 불량
 펜 불량: 팬은 좋은거 쓰자
 USB, ODD: 다른 USB로 시도
 RAM: window 7 이상은 거의 없음
 VGA: 칩셋 망가지면 사망선고
 콘덴서, 코일, FET(Field Effect Transister): CPU로 가는 높은 전압의 직류를 낮은 전압의 직류로 변환\

-OS 재설치는 주기적으로 하는게 좋음:
 공유파일이나 레지스트리 찌거기 정리됨

-PnP(Plug & Play): 드라이버 자동 설치
 외부 장치들은 매끄럽지 않음 -> 직접 설치

-드라이버 구하기:
 pc, 칩셋, 보드 제조사 홈페이지, 3dp chip, easy drv

-고장 비율:
 1. RAM:
    접촉 불량
 2. 전원장치(파워):
    콘댄서, 퓨즈 불량. 
    파워는 고쳐도 재발 많음. 중고 쓰지 말자
 3. 메인보드:
    칩셋 살짝 눌러서 내부 핀 크랙 문제인지 확인 -> 핀 문제면 교체
    IC(Integrated circuit, 직접 회로, 반도체 칩) 확인
    번개로 LAN카드 고장이면 LAN 카드 교체
 4. VGA:
    팬 고장은 자주 발생
 5. 모니터:
    할 수 있는거 없음
 6. CPU:
    소켓 파손, 핀 파손, 쿨러 고장: 쿨러는 저가 x

-청소는 10마력 정도 컴프레셔로. 이 물질 화재 위험, 쿨러 작동 방해

	5. 인터페이스

-드라이버 설치 확인:
 느낌표 있는지, 디스플레이: 표준 디스플레이면 안된거, 네트워크: 모델명 안나옴\

-보조기억장치 인터페이스:
 SATA3
 SATA3 M.2
 NVMe M.2
 M.2는 인터페이스의 폼 팩터(인터페이스의 외양적인 부분)

-NVMe(Non Volatile Memory express): PCI Express로 연결된 비휘발성 저장장치들을 위한 통신 프로토콜

-M.2 소켓:
 B키: 왼쪽 6핀 따로
 M키: 오른쪽 5핀 따로; NVMe SSD
 B+M키: 둘 다. SATA SSD

-PCIe: 직렬 연결 데이터 전송 레인(최대 16)으로 구성.

-USB2.0: 검정 소켓, 480Mbps // 이론 대역폭
 USB3.0: 파랑 소켓, 5Gbps
 USB3.1: 빨강 소켓, 10Gbps

-USB3.0을 2.0에 꽂으면 2.0속도
 USB2.0을 3.0에 꽂아도 2.0속도
